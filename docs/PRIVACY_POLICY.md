# ToDaDot 개인정보처리방침

**최종 업데이트: 2025년 12월 12일**

## 1. 개인정보의 수집 및 이용 목적

ToDaDot은 사용자의 개인정보를 수집하거나 외부 서버에 저장하지 않습니다.

## 2. 데이터 저장 및 처리

### 로컬 저장
- 모든 데이터(일정, 할 일 목록, 메모 등)는 사용자의 로컬 컴퓨터에만 저장됩니다.
- 데이터는 SQLite 데이터베이스 파일로 저장되며, 외부 서버로 전송되지 않습니다.

### Google API 연동 (선택사항)
- Google Calendar와의 동기화 기능을 사용하는 경우, Google OAuth 2.0을 통해 인증합니다.
- Google API를 사용하여 일정 데이터를 동기화하지만, 이는 Google의 서버와 직접 통신하며 ToDaDot 서버를 거치지 않습니다.
- Google OAuth 토큰은 사용자의 로컬 컴퓨터에만 저장되며, 외부 서버로 전송되지 않습니다.

### Firebase 연동 (선택사항)
- Firebase를 사용하여 일기 및 메모를 클라우드에 저장하는 기능이 있습니다.
- 이 기능은 사용자가 명시적으로 Firebase 설정을 입력한 경우에만 활성화됩니다.
- Firebase를 사용하는 경우, 데이터는 Firebase 서버에 저장되며 Google의 개인정보처리방침이 적용됩니다.

## 3. 개인정보의 제3자 제공

ToDaDot은 사용자의 개인정보를 제3자에게 제공하지 않습니다.

## 4. 개인정보의 보호

- 모든 데이터는 사용자의 로컬 컴퓨터에만 저장됩니다.
- Google API 또는 Firebase를 사용하는 경우, 해당 서비스의 보안 정책이 적용됩니다.

## 5. 사용자의 권리

사용자는 언제든지 앱을 삭제하여 모든 로컬 데이터를 제거할 수 있습니다.

## 6. 문의

개인정보처리방침에 대한 문의사항이 있으시면 GitHub 리포지토리의 Issues를 통해 문의해주세요.

---

**중요**: ToDaDot은 어떠한 사용자 데이터도 Google 외부 서버에 저장하지 않습니다. 모든 데이터는 사용자의 로컬 컴퓨터에만 저장되며, Google API 또는 Firebase를 사용하는 경우에만 해당 서비스의 서버와 통신합니다.
