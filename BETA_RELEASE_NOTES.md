## 📋 주요 기능

### 1. 아이젠하워 매트릭스 (Eisenhower Matrix)
- **중요도와 긴급도 기반 4분면 할 일 관리**
  - 중요하고 긴급한 일 (1사분면) Do
  - 중요하지만 긴급하지 않은 일 (2사분면) Plan
  - 중요하지 않지만 긴급한 일 (3사분면) Delegate
  - 중요하지도 긴급하지도 않은 일 (4사분면) Delete
- 드래그 앤 드롭으로 할 일 이동 및 순서 변경
- 날짜별 할 일 관리

### 2. 카테고리 기반 할 일 관리
- **사용자 정의 카테고리 생성 및 관리**
- 카테고리별 색상 커스터마이징
- 카테고리별 할 일 분류 및 필터링
- 카테고리 순서 재정렬

### 3. 캘린더 및 일정 관리
- **월간/주간 캘린더 뷰**
- Google Calendar API 연동
★★★ 해당 기능 현재 구글 심사 중이라 동작하지 않습니다 ★★★
- 일정 생성, 수정, 삭제
- 일정 카테고리별 색상 관리
- 일정과 할 일 통합 관리

### 4. Daily Focus (오늘의 집중)
- **오늘 집중할 할 일 선정**
- 시간 계획 패널 (Time Planner)
- Daily Focus 할 일 순서 조정
- 오늘 미완료 할 일 자동 이동 기능

### 5. 메모 및 일기 기능
- **날짜별 메모 작성**
- 리치 텍스트 에디터 (Rich Text Editor)
- 텍스트 포맷팅 (굵게, 기울임, 밑줄, 목록 등)
- 일기 작성 및 관리
- 메모/일기 검색 기능

### 6. 기분 추적 (Mood Tracker)
- **날짜별 기분 기록**
- 색상으로 기분 표현
- 캘린더에 기분 표시

### 7. 스티커 기능
- **캘린더 및 각 탭에 스티커 추가**
- 이미지 업로드 및 배치
- 스티커 크기 조절, 회전, 위치 이동
- 스티커 레이아웃 저장 및 불러오기
- 탭별 스티커 분리 관리

### 8. Firebase 클라우드 동기화

- **Firebase Authentication 로그인**
- 할 일 데이터 클라우드 동기화
- 다기기 간 데이터 동기화
- 로컬 → Firebase 자동 마이그레이션
- 실시간 양방향 동기화
★★★ 해당 기능 현재 구글 심사 중이라 동작하지 않습니다 ★★★

### 10. 커스터마이징 기능
- **헤더 이미지 설정**
- 테마 선택
- 카테고리 색상 커스터마이징
- 일정 색상 프리셋
- 스타일 관리 패널

### 11. 할 일 상세 기능
- **할 일에 메모 추가**
- 할 일 시간 설정

## ⚠️ 주의사항 및 알려진 제한사항

### 1. Beta 버전 특성
- **안정성**: Beta 버전이므로 일부 기능에서 예상치 못한 동작이 발생할 수 있습니다.
- **데이터 백업**: 중요한 데이터는 정기적으로 백업하시기 바랍니다.
- **피드백**: 버그나 개선사항은 개발자에게 알려주시면 감사하겠습니다.

### 2. Firebase 동기화
- **인터넷 연결 필요**: Firebase 동기화를 사용하려면 인터넷 연결이 필요합니다.
- **동기화 지연**: 네트워크 상태에 따라 동기화가 지연될 수 있습니다.
- **메모 저장**: 할 일의 메모는 SQLite에 별도 저장되며, Firebase에는 투두 객체의 `memo` 필드로 포함됩니다.
★★★ 해당 기능 현재 구글 심사 중이라 동작하지 않습니다 ★★★

### 3. Google Calendar 연동
- **OAuth 설정 필요**: Google Calendar 연동을 사용하려면 Google Cloud Console에서 OAuth 2.0 설정이 필요합니다.
- **API 할당량**: Google Calendar API의 일일 할당량 제한이 있습니다.
- **동기화 충돌**: 여러 기기에서 동시에 수정할 경우 충돌이 발생할 수 있습니다.
★★★ 해당 기능 현재 구글 심사 중이라 동작하지 않습니다 ★★★

### 4. 데이터 마이그레이션
- **최초 로그인 시**: Firebase 로그인 시 로컬 데이터가 자동으로 Firebase로 마이그레이션됩니다.
- **중복 방지**: 마이그레이션 플래그로 중복 마이그레이션을 방지합니다.
- **데이터 손실 위험**: 마이그레이션 중 데이터 손실을 방지하기 위해 로컬 백업을 유지합니다.
★★★ 해당 기능 현재 구글 심사 중이라 동작하지 않습니다 ★★★

### 5. 성능
- **대량 데이터**: 할 일이 매우 많을 경우 (수천 개 이상) 성능 저하가 발생할 수 있습니다.
- **스티커 이미지**: 큰 이미지 파일을 스티커로 사용할 경우 메모리 사용량이 증가할 수 있습니다.
- **동기화 시간**: 대량의 데이터를 동기화할 때 시간이 걸릴 수 있습니다.

### 6. 플랫폼 제한사항
- **Windows 전용**: 현재 Windows 플랫폼만 지원합니다.
- **Visual C++ 재배포 패키지**: 일부 시스템에서 Visual C++ 재배포 패키지 설치가 필요할 수 있습니다.

### 7. 보안 및 개인정보
- **Firebase 인증**: Firebase Authentication을 사용하여 사용자 인증을 처리합니다.
- **로컬 데이터**: 할 일 메모는 로컬 SQLite 데이터베이스에 저장됩니다.
- **Google OAuth**: Google 서비스 연동 시 OAuth 토큰이 로컬에 저장됩니다.

### 8. 기술 지원
- **버그 리포트**: 문제가 발생하면 개발팀에 알려주세요.
- **피드백**: 기능 개선 제안을 환영합니다.
